# NotebookLM 학습자료 생성 프롬프트 템플릿

## 개요

이 템플릿은 `notebooklm-inputs/` 의 배치 파일을 바탕으로 JLPT 학습자료를 생성할 때 사용합니다.

**워크플로우**:
1. `content/modules/notebooklm-inputs/{level}-module-{NNN}-batch-01.json` 파일 다운로드
2. 아래 프롬프트를 레벨에 맞게 선택하여 `[단어 목록]` 자리에 붙여넣기
3. NotebookLM / Claude / GPT-4o 등에 입력
4. 생성된 JSON을 관리자도구에 업로드 → 자동 커밋

---

| 레벨 | 장면 수 | 문장분석 수 | 문법 난이도 예시 |
|------|--------|-----------|----------------|
| N5   | 3장면   | 12-15개    | AはBです, ~ます, ~て |
| N4   | 4장면   | 15-18개    | ~ために, ~のおかげで, ~について |
| N3   | 4장면   | 16-20개    | ~ことになる, ~わけではない, ~にしても |
| N2   | 5장면   | 18-22개    | ~にもかかわらず, ~としても, ~をめぐって |
| N1   | 5장면   | 20-25개    | ~いかんによって, ~をもって, ~ならではの |

## 프롬프트 (N4용)

> **N5/N3/N2/N1은 장면 수·문법 목표만 위 표에 맞게 바꿔서 사용**

```
당신은 JLPT N4 일본어 학습 자료를 만드는 전문 교육자입니다.
아래 단어 목록을 활용하여 학습 자료를 JSON 형식으로 생성해주세요.

## 단어 목록 (25개)
[아래 형식으로 붙여넣기: word (read): mean]
예)
あいさつ (あいさつ): 인사
赤ちゃん (あかちゃん): 아기
あご (あご): 턱
...

## 생성 지침

### 1. story (HTML 문자열)
- **4개 장면** 구성: <h3>Scene N. 일본어 제목 (한국어)</h3><p>내용</p>
- 장면당 4-6문장, 일상적인 상황을 배경으로 함
- 입력 단어의 **80% 이상** 자연스럽게 사용
- **모든 한자**에 <ruby>漢字<rt>よみ</rt></ruby> 태그 필수
- N4 수준의 문법·어휘만 사용 (N3 이상 표현 금지)
- 중점 문법: ~ために, ~のおかげで, ~について, ~みたいに, ~として

### 2. analysis (배열, 15-18개 항목)
이야기에서 핵심 문장을 추출하고 각 항목을 아래 형식으로 작성:
- sent: 원문 문장 (HTML ruby 태그 포함)
- trans: 한국어 번역
- grammar: 사용된 문법 패턴명 (한국어)
- tags: 해당 문장에 등장한 입력 단어 목록 (배열)

### 3. quiz (배열, 10문항)
구성: 읽기(よみ) 4문항 + 의미 3문항 + 문장완성 3문항
각 항목 형식:
- q: 질문 (일본어)
- opt: 선택지 4개 배열 ["A", "B", "C", "D"]
- ans: 정답 인덱스 (0-3 정수)
- comment: 한국어 해설 (정답 단어의 의미·사용법 포함)

문제 유형 예시:
- 읽기: 「赤ちゃん」の読み方は何ですか。
- 의미: 「안도」를 일본어로 쓰면 무엇입니까。
- 문장완성: 電車が遅れた______、会議に遅刻しました。(答: ために)

## 출력 형식

반드시 **JSON만 출력** (설명 텍스트 없이):

{
  "story": "<h3>Scene 1. ...</h3><p>...</p><h3>Scene 2. ...</h3><p>...</p><h3>Scene 3. ...</h3><p>...</p><h3>Scene 4. ...</h3><p>...</p>",
  "analysis": [
    {
      "sent": "<ruby>今日<rt>きょう</rt></ruby>は<ruby>会議<rt>かいぎ</rt></ruby>があります。",
      "trans": "오늘은 회의가 있습니다.",
      "grammar": "~があります (존재/예정 표현)",
      "tags": ["会議"]
    }
  ],
  "quiz": [
    {
      "q": "「赤ちゃん」の読み方は何ですか。",
      "opt": ["あかちゃん", "あかこ", "あかさん", "あかぼう"],
      "ans": 0,
      "comment": "赤ちゃん(あかちゃん)은 '아기'라는 뜻입니다."
    }
  ]
}
```

## 출력 품질 체크리스트

생성된 JSON을 관리자도구에 업로드하기 전 확인:

- [ ] story에 4개 장면이 모두 있는가
- [ ] 모든 한자에 ruby 태그가 있는가
- [ ] 입력 단어의 80% 이상이 story에 등장하는가
- [ ] analysis 항목이 15개 이상인가
- [ ] quiz가 정확히 10문항인가
- [ ] quiz의 opt가 항상 4개인가
- [ ] ans가 0-3 범위 정수인가
- [ ] JSON이 올바르게 파싱되는가 (온라인 JSON 검증기 활용)

## 레벨별 프롬프트 차이점 요약

### N5용 변경 사항
- 장면 수: **3개** 장면
- 문법: N5 수준 (AはBです, ~ます형, ~て형, 기본 조사)
- analysis: 12-15개
- 문법 언급: ~です/ます, ~ている, ~てください

### N3용 변경 사항
- 장면 수: **4개** 장면
- 문법: N3 수준 (~ことになる, ~わけではない, ~にしても)
- analysis: 16-20개

### N2용 변경 사항
- 장면 수: **5개** 장면
- 문법: N2 수준 (~にもかかわらず, ~としても, ~さえ~ば)
- analysis: 18-22개
- 더 복잡한 사회적/추상적 주제 사용

### N1용 변경 사항
- 장면 수: **5개** 장면
- 문법: N1 수준 (~いかんによって, ~をもって, ~ならではの)
- analysis: 20-25개
- 격식체 및 문어체 표현 포함 가능
